# MATLAB Line-Following Minidrone

An end-to-end autonomous flight simulation framework leveraging MATLAB and Simulink. This repository implements a robust line-following navigation pipeline for a virtual minidrone, integrating computer vision, real-time image processing, and closed-loop PID control system design.

The architecture is built upon the official **MATLAB/Simulink Minidrone Competition** template, featuring configurable linear and non-linear airframe variants for multi-fidelity flight dynamics testing.

---

## 📂 Repository Structure

The project utilizes the standard MATLAB Project (`.prj`) structure to manage dependencies, paths, and environment shortcuts seamlessly:

* **`mainModels/`**: Contains the primary Simulink models (`.slx`) used to orchestrate and run the full flight simulations.
* **`controller/`**: Core Flight Control System (FCS) algorithms, including the vision-based line detection logic and cascading PID control loops.
* **`linearAirframe/`**: Linearized plant models representing simplified drone dynamics, ideal for rapid control law design and frequency-domain analysis.
* **`nonlinearAirframe/`**: High-fidelity, non-linear airframe plant models capturing complex physical dynamics, environmental disturbances, and aerodynamic effects.
* **`tasks/`**: Setup scripts, initialization parameters, and design variable definitions executed automatically during environment boot.
* **`utilities/`**: Automation utilities, plotting scripts, and operational helper functions.
* **`support/` & `libraries/`**: Required custom blocks, external hardware definitions, and shared functional dependencies.
* **`MinidroneCompetition.prj`**: The master MATLAB Project configuration file. Opening this automatically sets up the search paths and loads variables into the workspace.

## 🛠️ Prerequisites & Installation

To run the simulations successfully, ensure your host environment meets the software and toolbox requirements.

### Required Software & Toolboxes
* **MATLAB** (R2020a or newer recommended)
* **Simulink**
* **Aerospace Blockset**
* **Computer Vision Toolbox**
* **Control System Toolbox**
* **Stateflow**

### Getting Started

1. **Clone the Repository:**
   ```bash
   git clone [https://github.com/ArshithaRajkumar/Matlab-Line-Following-Minidrone.git](https://github.com/ArshithaRajkumar/Matlab-Line-Following-Minidrone.git)
   ```

2. **Launch MATLAB:** Open MATLAB and navigate directly to the root directory of the cloned repository.

3. **Initialize the Environment:**
   Double-click and open the `MinidroneCompetition.prj` file. This action will automatically execute startup scripts, instantiate variables in the workspace, and load specific toolstrip shortcuts.

## 🚀 How to Run the Simulation

1. **Workspace Setup:** Ensure the project environment is active (via the `.prj` initialization). If variables need to be reloaded manually, run the setup scripts located in the `tasks/` folder.
2. **Open the Core Model:** Navigate to the `mainModels/` folder and open the primary simulation wrapper file (e.g., `ctrl_main.slx`).
3. **Configure the Plant Fidelity:** Toggle the variant subsystem block to choose between `linearAirframe` (for rapid tuning) or `nonlinearAirframe` (for real-world physical verification).
4. **Execute Flight:** Click the **Run** button in the Simulink toolstrip.
5. **Analyze Results:** Observe the flight profile using the integrated 3D visualization tool alongside time-domain scope blocks displaying lateral tracking errors and control effort outputs.

## 🧠 System Architecture & Control Theory

The minidrone's autonomy loop operates continuously across two primary components managed within the `controller/` subsystem:

### 1. Computer Vision Pipeline
* **Acquisition:** The vehicle's downward-facing camera captures real-time frame sequences of the ground track.
* **Segmentation:** The processing pipeline binarizes the images to isolate the track markers from background noise.
* **Error Extraction:** The algorithm extracts critical geometric metrics, computing the drone's **lateral offset** (distance from the track centerline) and **heading error** (angular deviation relative to the path direction).

### 2. Flight Control System (FCS)
* **Inner Loop:** High-frequency stabilization handling raw roll, pitch, yaw rate, and altitude dynamics.
* **Outer Loop:** Processes spatial error inputs generated by the vision pipeline. Tuned PID controllers translate these path errors into precise attitude targets, driving the minidrone smoothly along the line topology.
